# 极简版「主动预约 + 被动提醒」完整逻辑（毕设专用）

## 一、前置基础：儿童信息与疫苗的**极简绑定**（核心中的核心，最简化）

这是提醒和预约能触发的基础，不用做国家免疫规划的复杂规则，只做**简单年龄匹配**即可：

### 1. 家长添加儿童时：**只填出生日期（必填）**

- 小程序「添加儿童」页面，**出生日期设为必填项**（姓名、性别也必填，其他身份证、地址全设为可选）；
- 系统接收到出生日期后，**只做一个简单计算**：`当前年龄（周岁）= 今年 - 出生年`（不用算月龄、天，毕设演示完全够用，开发量骤降）。

### 2. 后台疫苗管理：**为每个疫苗设置「可接种年龄」（固定值，管理员手动填）**

- 后台添加疫苗时，加一个 **「可接种年龄」输入框 **（比如乙肝疫苗填 0，脊灰疫苗填 1，水痘疫苗填 1 岁，百白破填 2 岁）；
- 不用做剂次、间隔，一个疫苗只对应一个年龄，比如 “脊灰疫苗 - 1 岁可接种”，简单直接。

**举个例子**：

家长添加儿童，出生年 2023 年，当前 2025 年 → 系统计算年龄 2 岁；

后台脊灰疫苗可接种年龄设为 1 岁 → 这个儿童**符合脊灰疫苗接种条件**，可预约、到期会提醒。

## 二、极简版「家长主动预约」逻辑（3 步走完，无复杂校验，符合基础生活逻辑）

### 核心目标：家长能选孩子、选疫苗、提交预约，管理员能审核，状态能流转即可。

#### 1. 家长端操作（3 步，超简单）

```
打开小程序 → 儿童管理选孩子 → 疫苗列表选疫苗（系统只做1个校验：孩子年龄≥疫苗可接种年龄，不符合直接提示）→ 选接种日期（直接让家长选任意未来日期，不用限制开放日/库存）→ 提交预约
```

**砍掉的复杂点**：

- 不限制预约日期（不用后台设置开放日，家长想选哪天选哪天）；
- 不校验疫苗库存（库存只做后台展示，不影响预约提交）；
- 不限制重复预约（毕设演示基本不会出现，不用做）；
- 取消预约：只做「审核通过前可取消」，审核通过后不能取消（不用加 24 小时限制）。

#### 2. 后台管理端操作（2 步）

```
管理员登录后台 → 预约管理看到「待审核」列表 → 点击「通过」/「拒绝」（拒绝时只填简单原因，比如“暂不接种”）→ 审核结果同步到小程序家长端
```

**预约状态只保留 3 个（砍掉 1 个，最简化）**：

`待审核` → 管理员操作 → `已通过` / `已拒绝`（砍掉「已完成」，接种完成状态可让管理员手动标记，演示时提一句即可，不用做流程）。

#### 3. 系统后台处理（无复杂逻辑，只做数据存储）

家长提交预约 → 系统校验「孩子年龄≥疫苗可接种年龄」→ 校验通过存数据库（状态待审核）→ 管理员审核后更新状态 → 小程序端刷新展示最新状态。

## 三、极简版「系统被动提醒」逻辑（2 大场景，定时任务最简化，短信模拟发送）

这是毕设的**亮点功能**，但实现起来超简单，**核心就是「定时任务扫描数据库 + 模拟发短信」**，不用对接任何第三方平台，不用做防重复提醒，演示效果拉满。

### 提醒只做 2 个核心场景（砍掉所有额外提醒，聚焦重点）

#### 场景 1：**年龄到期提醒**（孩子年龄到了疫苗可接种年龄，提醒家长预约）

- **触发条件**：孩子当前年龄 = 某疫苗的「可接种年龄」（比如孩子 2 岁，疫苗可接种年龄 2 岁，就触发）；
- **触发时机**：每天只扫描 1 次，只要满足条件就提醒（不用做 7 天再提醒，不用防重复）；
- **提醒内容**：固定模板，替换孩子名、疫苗名即可。

#### 场景 2：**预约后提醒**（家长预约审核通过后，提前 3 天 + 1 天提醒接种）

- **触发条件**：预约状态为「已通过」，且距离接种日期还有 3 天 / 还有 1 天；
- **触发时机**：每天扫描 1 次，满足哪个条件就发哪个提醒；
- **提醒内容**：固定模板，替换孩子名、疫苗名、接种日期即可。

### 技术实现：Spring Task 定时任务（5 行核心代码，毕设超友好）

#### 1. 定时任务只做**1 个**（砍掉多个时间点，每天凌晨 1 点执行一次，扫描所有提醒）

- 不用分凌晨扫到期、下午扫预约，一个定时任务全搞定，代码少一半；
- 启动类加`@EnableScheduling`，任务类加`@Scheduled(cron = "0 0 1 * * ?")`，完事。

#### 2. 短信发送：**纯模拟（毕设演示首选，不用申请任何接口）**

这是最关键的简化！不用对接阿里云 / 腾讯云短信，不用配 apikey，不用花钱，**3 种模拟方式任选，演示效果和真实的一样**：

- 方式 1（最推荐）

  ：控制台打印 + 数据库存记录

  - 定时任务触发提醒后，控制台打印：`【模拟短信】发送至13800138000：您好，您的孩子小明已满2岁，可预约百白破疫苗，请尽快登录小程序预约~`；
  - 新建一张简单的`remind_log`表（只存：孩子名、手机号、提醒内容、发送时间），把提醒信息存进去，后台加一个「提醒日志」页面，答辩时打开给老师看，直观又专业。

  

- 方式 2

  ：小程序站内信

  - 小程序加一个「消息中心」页面，提醒信息直接推送到这里，家长点开就能看，不用发短信。

  

- 方式 3

  ：本地邮件测试（可选，稍微复杂一点）

  - 用 Spring Mail 对接自己的 QQ/163 邮箱，把提醒发送到自己的邮箱，演示时打开邮箱给老师看。

  

#### 3. 扫描逻辑（极简 SQL，不用复杂计算）

- 年龄到期提醒扫描

  ：

  

  查所有儿童，计算年龄（今年 - 出生年），匹配疫苗表中「可接种年龄 = 儿童年龄」的疫苗，再查这个孩子是否没预约过该疫苗，满足就生成提醒。

- 预约后提醒扫描

  ：

  

  查所有「已通过」的预约记录，计算「接种日期 - 当前日期」的天数，等于 3 或等于 1，就生成提醒。

**砍掉的复杂点**：

- 不用做提醒去重（毕设演示不会重复触发，就算重复了，控制台多打印一条也不影响）；
- 不用做短信模板配置（后台不用加模板页面，代码里写死 2 个模板即可）；
- 不用做提醒失败重发（模拟发送不会失败，不用处理）。
